# Copilot Instructions for `nba-predictions`

## Purpose
Provide guidance for AI-assisted updates to data ingestion, model pipeline, and Streamlit UI in this repository.

## High-level behavior
- Prefer verified source fields over assumed column names.
- Always include code-based validation (e.g., `pd.read_parquet(...).columns`) before changing column mappings or UI labels.
- Keep data source precedence explicit: `nba_official` referee assignments => ESPN fallback ONLY for missing games.
- Keep regression cases covered in README changelog.

## Key files and responsibilities
- `scripts/scrape_external.py`: referee ingestion from nba_official / ESPN.
- `scripts/preload_cache.py`: pre-warm historical parquet cache, refresh hoopR data, and run predictions.
- `scripts/fetch_hoopr_data.py`: download sportsdataverse hoopR team box / player box and aggregate PBP features.
- `utils/hoopr_fetcher.py`: disk-cached hoopR parquet loader and season mapping.
- `utils/data_fetcher.py`: data layer cache, prediction retrieval, injury report parsing.
- `pages/1_Game_Predictions.py`: main prediction UI and injury table.
- `pages/3_Team_Stats.py`: standings and team metrics display.
- `pages/4_Player_Stats.py`: player game log display.
- `.github/workflows/referee-assignments.yml`: daily referee refresh schedule.
- `.github/workflows/nightly-pipeline.yml`: nightly data pipeline.
- `.github/workflows/hoopr-daily.yml`: scheduled hoopR data refresh and committed parquet caching.

## Referee ingestion and deduplication
- `nba_official` is authoritative for `NBA_GAME_ID` / team names.
- ESPN data is a fallback for games missing from official page, not used for same-game overwrite.
- Referee output schema should include: `NBA_GAME_ID`, `HOME_TEAM`, `AWAY_TEAM`, `HOME_TEAM_FULL`, `AWAY_TEAM_FULL`, `REFEREE`, `ORDER`, `CALLED_FOULS_PER_GAME`, `HOME_WIN_PCT`, `FOUL_PCT_ROAD`, `FOUL_PCT_HOME`, `FOUL_DIFFERENTIAL`, `EXPERIENCE_YEARS`, `FETCH_DATE`, `SOURCE`.

## Injury report
- Source: `https://site.api.espn.com/apis/site/v2/sports/basketball/nba/injuries`
- Use `team_entry.get('displayName', '')` for team, not `team_entry.get('team', {}).get('displayName', '')`.
- Output columns: `team`, `player_name`, `position`, `status`, `description`.

## Workflow drift guards
- Ensure scheduled times match Eastern (ET) requirements.
- Add `workflow_dispatch` hook with optional `force` param for manual override.

## Verification steps for PR reviews
1. Run `python -c "import pandas as pd; ..."` over relevant parquets to confirm field names before UI renames.
2. Check `get_injury_report()` returns non-empty `team` fields.
3. Confirm `utils/hoopr_fetcher.py` can load disk-cached hoopR parquets without network access.
4. Ensure `scripts/fetch_hoopr_data.py --skip-pbp` and `scripts/preload_cache.py` run end-to-end.
5. Confirm `.github/workflows/hoopr-daily.yml` is set to update and commit `data_files/hoopr/` parquets.
6. Run `python -m pytest` or `get_errors` to verify no linting/type issues.

## Notes for reviewers
- Correctness > minimal scope; include fallback behavior explicitly.
- Always update `README` with major pipeline changes and bugfixes.
- Keep column naming in UI consistent across pages and with user expectation (title case).
